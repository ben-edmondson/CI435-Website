Week 4: October 29, 2020

This is the first week we begun to start learning the basics of CSS. We first learned how the 'normalize.css' file is used to standardise webpages. Which is used by most professional developers. As such, in order to follow this standard, this was used on this and subsequent pages within this website.

After this we also added 'stylesheet.css' to our CSS folder. This is how the bulk of visual changes within a page are made. For example, all of the colours you see within this page were changed within the stylesheet.

We used this week to learn basic styling, such as changing font style, size, and colour. We also learned about hexcodes. Hexcodes are how the browser identifies different colours and applies them.

Week 6: November 12, 2020

This week I taught myself how to use 'form' tag. I did this in order to allow for a user to send a message from the Contact Me page. A form has many nested elements within it in order to make it work. Without them it is effectively useless. It does this by using 'label' and 'input' tag. A label tag is used in order to label the input and the input tag is used to input data. Fairly simple stuff. Finally you can create a 'submit' button. in the exact same way you do an input or label except using the submit tag instead.

Week 7: November 19, 2020

This week I learned about CSS grids and design. This begun with box models. The box model is a description of the area taken up by an element. A box has 4 different edges to consider:

  • margin
  • border
  • padding
  • content

A box model effectively wraps around every single element in HTML. When designing and styling an element. it is important to consider the box model. A good example of where the box model was considered would be each subheading for each week. Understanding the box model played a key part in creating the design for it.

Week 8: November 26, 2020

This week I begun to learn about media queries. At first I did not really understand how they worked at all. There was a lot of struggle in understanding what they did and how they worked. This was because my website design had become far too complex to add media queries with ease. I learned that using a design principle of "mobile" first would be the correct approach. With this in mind I begun to redesign my entire website with something more simple in mind.

Week 9: December 3, 2020

This week I finally figured out how media queries work. I used a few websites which are linked in the references section in order to gain a better understanding of their function and complexities. This allowed me to understand that a media query is effectively an IF statement. This means that the query was checking to see if the screen size was correct to the query and would change the website according to that. With that in mind I begun to add simple queries. First fixing the navigation bar so that it would display differently if on a smaller screen. Then I worked on removing the sidebar I had added if the margin was not big enough to fit it in. After doing this I realised I was still having some issues however. I was unable to figure it out at first until I realised that using px as a unit of measurement was not a good idea. at this point I converted all my measurements to em. This is because pixels are not a good relative unit of measurement whereas em is. I also had images removed from the Learning Journal on mobile screens. This is because I believed the page looked better without them.

Week 10: December 10, 2020

This week I begun testing the website extensively. First I began with validation of pages. This had no issues fortunately as I had continually validated the website while working on it. This ensured a very short process. Next I begun with testing how pages worked in different views. This included mobile and smaller sizes on desktop. Once again this worked as intended entirely. Because of testing in practice every time I had a new addition there were no errors to fix. The only changes that were needed were stylistic choices that I believed would improve the look of the website.
